# 智能财务薪酬集团版 Demo

## 打开方式

在浏览器中打开 `index.html`。如果页面已经打开，请刷新标签页加载最新版本。

## 推荐演示路径

1. 在“工作台”查看资料准备、算薪、个税、工资条确认、社保月结和成本分析六阶段流程。
2. 在“薪资计算 → 薪资档案”查看员工所属公司、报税主体、参保主体和产品线分摊，点击“编辑分摊”体验 100% 校验。
3. 在“薪资计算 → 个税计算明细（全年）”点击“全年明细”，查看 1–12 月收入、扣除和个税。
4. 在“个税管理 → 人员信息报送”模拟重新报送失败人员。
5. 在“工资条管理 → 员工确认记录”模拟员工登录并确认工资条。
6. 在“社保公积金 → 月结账单”模拟核对账单差异。
7. 在“报表中心 → 成本看板”切换所属公司和产品线成本维度。
8. 在“系统设置”查看集团公司、产品线分摊规则和用户权限。
9. 在“薪资计算 → 工作台”点击“一键计算全部公司”，并展开集团、公司和员工三级明细。
10. 在“社保公积金 → 参保人员”编辑员工月度社保及公积金基数，体验超限预警和自动重算。
11. 在“薪资计算 → 工作台”模拟导入绩效分，或在员工明细中手工编辑绩效分。

## 业务规则

- 每名员工在一个月份仅归属一家所属公司。
- 所属公司、报税主体和参保主体强制一致；员工档案只维护所属公司。
- 员工薪酬成本可以分摊到多个产品线，比例合计必须为 100%。
- 个税累计维度为员工、报税主体和自然年度。
- 发薪存证以员工登录员工端确认工资条的记录为核心。

## 数据说明

- Demo 使用 3 家虚构公司、4 条虚构产品线和 6 名虚构员工。
- 个税专项附加扣除采用中国统一政策口径，住房租金展示上海每月 1,500 元档位。
- 员工参保地统一为上海，社保与公积金基数按政策快照的上下限校验。
- 社保基数与公积金基数可分别手工录入；超出上海上下限仅提示风险，仍按录入值计算。
- 基数修改需填写生效月份和原因，并模拟生成后续月份快照及新算薪版本。
- 薪资档案保存基本工资、绩效工资基数和入职信息；绩效分按月导入或手工录入。
- 最终绩效工资等于绩效工资基数乘以当月绩效分。
- 上海年度社保、公积金参数标记为“Demo 待复核”，正式启用前必须按官方年度通知复核。
- 公司税号、人员姓名、金额、银行卡尾号、报送状态和确认记录均为虚构测试数据。
- 所有“报送、发布、确认、核对和导出”操作均为页面模拟，不连接真实外部系统。
- 当前版本不可用于正式发薪、纳税申报或社保公积金缴纳。
